# nload 详细教程（运维实战版）

## 一、nload 是什么（先摆正位置）

一句话定位：

> **nload = 网卡级别实时吞吐量监控工具**

它解决的问题是：

- 网卡现在 **有没有跑满？**
- 入口 / 出口流量大不大？
- 是 **持续高流量** 还是 **瞬时波动**？

❌ 看不了 IP
❌ 看不了端口
❌ 看不了进程

👉 **它就是“网络体温计”**

------

## 二、nload 的工作方式

- 从 `/proc/net/dev` 读取网卡统计
- **几乎无性能损耗**
- 不抓包、不解析协议
- 非常适合 **生产环境长期看**

------

## 三、最基本用法（必会）

### 1️⃣ 直接启动

```bash
nload
```

效果：

- 自动检测所有网卡
- 单个界面显示
- 可用方向键切换

------

### 2️⃣ 指定网卡（推荐）

```bash
nload eth0
```

多网卡：

```bash
nload eth0 ens33
```

------

## 四、界面字段逐项精讲（核心）

```
Incoming:
Curr:  12.34 MBit/s
Avg:   8.21 MBit/s
Min:   0.00 MBit/s
Max:   98.20 MBit/s
Ttl:   123.4 GByte
```

### 字段说明

| 字段      | 含义                     |
| --------- | ------------------------ |
| Curr      | 当前速率                 |
| Avg       | 平均速率（由 `-a` 控制） |
| Min / Max | 运行以来最小 / 最大      |
| Ttl       | 启动后累计流量           |

下方曲线：

- 越接近顶部 → 越接近带宽上限

------

## 五、命令行参数逐个拆解（按使用频率）

------

### 🔹 `-t interval`（刷新间隔）

```bash
nload -t 200
```

- 单位：毫秒
- 默认：500ms

📌 建议：

- 排障：`200`
- 常规监控：`500~1000`

------

### 🔹 `-a period`（平均值时间窗）

```bash
nload -a 60
```

- 单位：秒
- 默认：300 秒

📌 场景：

- 看短期抖动 → 60
- 看平稳趋势 → 300

------

### 🔹 `-u`（速率单位）

```bash
nload -u m      # MBit/s
nload -u M      # MByte/s
```

| 参数  | 含义         |
| ----- | ------------ |
| h / H | 自动         |
| b / B | Bit / Byte   |
| k / K | kBit / kByte |
| m / M | MBit / MByte |
| g / G | GBit / GByte |

👉 **带宽对外沟通一般用 Bit，运维内部看 Byte**

------

### 🔹 `-U`（总流量单位）

```bash
nload -U G
```

控制 `Ttl` 显示单位。

------

### 🔹 `-i / -o`（图形刻度上限）

```bash
nload -i 1000 -o 1000
```

含义：

- 入站 / 出站 **100% = 1000 kBit/s**

📌 典型配置：

- 1Gbps 网卡 → `1000000`
- 100Mbps → `100000`

------

### 🔹 `-m`（多网卡模式）

```bash
nload -m
```

效果：

- 同屏显示多个网卡
- **不显示曲线**
- 只看数字

👉 **服务器多网卡首选**

------

## 六、运行时配置（F2）

运行中按：

```text
F2 → Options
```

可以实时修改：

- 刷新间隔
- 单位
- 刻度
- 平均窗口

👉 调好后 `F5` 可保存

------

## 七、典型实战用法（直接抄）

### 🚨 场景 1：服务器“感觉很慢”

```bash
nload eth0
```

判断：

- Curr 接近带宽上限 → 网络瓶颈
- Curr 很低 → 看 CPU / IO / 应用

------

### 🚨 场景 2：是否长期跑满？

```bash
nload -a 600 eth0
```

看 `Avg`：

- 长期高 → 扩容 / 限流
- 偶发高 → 正常波动

------

### 🚨 场景 3：多网卡服务器

```bash
nload -m -u m
```

快速判断：

- 哪块网卡在扛流量

------

## 八、nload + iftop 标准组合拳（重点）

### 运维黄金流程

1️⃣ **nload**

```bash
nload eth0
```

→ 判断是不是网络问题

2️⃣ **iftop**

```bash
iftop -i eth0 -n -N -P
```

→ 判断谁在占带宽

3️⃣ **ss / 应用**

```bash
ss -antp
```

------

## 九、nload 的优缺点（别用错）

### 优点

- 极低开销
- 简单直观
- 生产环境安全

### 缺点

- 看不到细节
- 不能定位 IP / 进程

👉 **它永远是第一步，不是最后一步**

------

## 十、一句话总结

> **nload = 判断“网络有没有问题”的最快方式**
