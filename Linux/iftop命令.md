`iftop` 是 Linux 下一个**实时查看网络流量**的神器，偏向「**谁在跟谁通信、占了多少带宽**」，非常适合排查带宽异常、流量突增、被谁拖慢网速这类问题 👀

------

## 一、iftop 是干嘛的？

简单说一句话：

> **iftop = 网络版 top**（看的是网卡流量，而不是 CPU）

它可以实时显示：

- 本机 ↔ 对端 IP / 主机
- 每个连接的 **实时带宽**
- 最近 **2s / 10s / 40s** 的平均流量
- 总发送 / 接收流量

非常适合你们运维/排障场景用。

------

## 二、安装 iftop

### CentOS / Rocky / Alma

```bash
yum install -y iftop
```

### Ubuntu / Debian

```bash
apt update
apt install -y iftop
```

⚠️ 需要 **root 权限** 才能抓包。

------

## 三、最常用的启动方式

### 1️⃣ 直接启动（默认网卡）

```bash
iftop
```

### 2️⃣ 指定网卡（强烈推荐）

```bash
iftop -i eth0
```

> 不确定网卡名？先：

```bash
ip a
```

------

## 四、界面怎么看？（核心）

启动后你会看到类似：

```
TX:             RX:
1.23Mb          3.45Mb

192.168.1.10  => 10.0.0.5      500Kb  450Kb  300Kb
               <=              200Kb  180Kb  150Kb
```

### 字段说明

| 项目           | 含义                        |
| -------------- | --------------------------- |
| `=>`           | 本机 **发出** 流量          |
| `<=`           | 本机 **接收** 流量          |
| 2s / 10s / 40s | 最近 2 / 10 / 40 秒平均速率 |
| TX / RX        | 总发送 / 接收               |

------

## 五、运行中常用快捷键（🔥重点）

进入 iftop 后直接按键即可：

| 按键 | 作用                 |
| ---- | -------------------- |
| `q`  | 退出                 |
| `h`  | 帮助                 |
| `n`  | 显示端口号           |
| `p`  | 显示端口             |
| `P`  | 按端口聚合           |
| `t`  | 显示 TX / RX / TOTAL |
| `s`  | 是否显示源地址       |
| `d`  | 是否显示目标地址     |
| `N`  | IP ⇄ 主机名切换      |
| `T`  | 显示累计流量         |
| `L`  | 设定流量过滤阈值     |

👉 **排查带宽问题必按：`n`、`p`、`t`、`T`**

------

## 六、非常实用的参数组合（运维常用）

### 1️⃣ 只看某个端口（如 MySQL）

```bash
iftop -i eth0 -f "port 3306"
```

### 2️⃣ 只看某个 IP

```bash
iftop -i eth0 -f "host 10.0.0.5"
```

### 3️⃣ 看整个子网

```bash
iftop -i eth0 -f "net 192.168.1.0/24"
```

### 4️⃣ 不解析主机名（更快）

```bash
iftop -n
```

### 5️⃣ 以字节显示（不是 bit）

```bash
iftop -B
```

------

## 七、iftop vs 其他工具（选哪个？）

| 工具         | 看什么           | 适合场景             |
| ------------ | ---------------- | -------------------- |
| iftop        | 连接 / IP / 端口 | **实时排查谁占带宽** |
| nload        | 总体流量         | 看网卡是否跑满       |
| iptraf-ng    | 详细协议         | 深度网络分析         |
| ss / netstat | 连接状态         | 端口 / 连接数        |

👉 **带宽异常第一反应：iftop**

## 八、典型排障场景（你一看就懂）

### 🚨 “服务器突然卡、带宽跑满”

```bash
iftop -i eth0 -n -P
```

看：

- 哪个 IP
- 哪个端口
- 是不是某个服务在疯狂发数据
