## 一、Linux 常见文件系统类型总览

| 文件系统         | 主要用途        | 是否日志 | 特点               |
| ---------------- | --------------- | -------- | ------------------ |
| **ext4**         | 默认通用        | ✅        | 稳定、兼容性最好   |
| **xfs**          | 大数据 / 大文件 | ✅        | 高性能、扩展快     |
| **ext3**         | 老系统          | ✅        | 已基本淘汰         |
| **ext2**         | 启动/小盘       | ❌        | 简单、无日志       |
| **btrfs**        | 新一代          | ✅        | 快照、压缩         |
| **zfs**          | 企业存储        | ✅        | 高可靠、需额外安装 |
| **swap**         | 虚拟内存        | ❌        | 内存交换           |
| **vfat / fat32** | U盘/EFI         | ❌        | 跨平台             |
| **ntfs**         | Windows 盘      | ❌        | 需 ntfs-3g         |
| **iso9660**      | 光盘            | ❌        | 只读               |
| **tmpfs**        | 内存盘          | ❌        | 基于 RAM           |

------

## 二、最常见的三种（重点）

### ⭐ ext4（默认王者）

- 大多数 Linux 发行版默认
- 最大单文件：**16TB**
- 支持日志、延迟分配
- **最稳妥**

📌 适合：

- `/`
- `/boot`
- 普通服务器

------

### ⭐ xfs（性能王）

- RedHat / CentOS / Rocky 默认
- 特别适合 **大文件、高并发**
- 不支持在线缩容（⚠️）

📌 适合：

- `/data`
- 日志、监控、对象存储
- Kafka / ES / SkyWalking

------

### ⭐ swap（不是文件系统）

- Linux 虚拟内存
- 可为：
  - swap 分区
  - swap 文件

```bash
lsblk -f
sda2 swap [SWAP]
```

------

## 三、特殊 / 虚拟文件系统

### 🔹 tmpfs（内存文件系统）

```bash
df -h | grep tmpfs
```

特点：

- 存在内存里
- 重启丢失
- 速度极快

常见挂载点：

- `/run`
- `/dev/shm`

------

### 🔹 proc / sysfs（内核接口）

```bash
mount | grep -E "proc|sysfs"
```

| 类型  | 作用           |
| ----- | -------------- |
| proc  | 进程、内核信息 |
| sysfs | 设备信息       |

## 四、如何查看文件系统类型（常用命令）

### 1️⃣ lsblk（推荐）

```bash
lsblk -f
```

### 2️⃣ df

```bash
df -Th
```

### 3️⃣ blkid

```bash
blkid
```

------

## 五、如何创建不同文件系统

### ext4

```bash
mkfs.ext4 /dev/sdb1
```

### xfs

```bash
mkfs.xfs /dev/sdb1
```

### swap

```bash
mkswap /dev/sdb2
swapon /dev/sdb2
```

## 六、运维选型建议（经验）

| 场景         | 推荐  |
| ------------ | ----- |
| 系统盘       | ext4  |
| 数据盘       | xfs   |
| Docker / K8s | xfs   |
| 数据库       | xfs   |
| 小分区       | ext4  |
| 临时数据     | tmpfs |
