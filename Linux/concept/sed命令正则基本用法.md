`sed` 的强大之处其实就在 **正则表达式（regex）**。掌握 sed 正则后，基本可以完成 **日志分析、配置修改、文本清洗、接口日志处理** 等很多工作。

我给你整理一份 **程序员常用 sed 正则使用指南**，都是实际工作中经常用到的。

------

# 一、sed 正则基本用法

最常见格式：

```bash
sed 's/正则/替换内容/' 文件
```

例子：

```bash
echo "user=tom" | sed 's/user/admin/'
```

输出

```
admin=tom
```

------

# 二、sed 常用正则符号

| 正则     | 含义                    |
| -------- | ----------------------- |
| `^`      | 行开头                  |
| `$`      | 行结尾                  |
| `.`      | 任意字符                |
| `*`      | 前一个字符重复0次或多次 |
| `+`      | 重复1次或多次           |
| `?`      | 可选                    |
| `[abc]`  | 匹配a或b或c             |
| `[0-9]`  | 数字                    |
| `[^abc]` | 不包含abc               |
| `()`     | 分组                    |
| `\1`     | 引用分组                |

注意：

sed 默认是 **基本正则 BRE**

如果要使用 `+ ? ()`

需要：

```bash
sed -E
```

或

```bash
sed -r
```

------

# 三、最常用的 sed 正则示例

## 1 匹配行开头

删除以 `#` 开头的行

```bash
sed '/^#/d' file.txt
```

示例

```
# comment
hello
```

输出

```
hello
```

------

## 2 匹配行结尾

删除以 `.log` 结尾的行

```bash
sed '/\.log$/d' file.txt
```

注意

`.` 需要转义

```
\.log
```

------

# 四、匹配任意字符

`.` 表示任意字符

示例：

```bash
echo "abc123" | sed 's/a.c/xxx/'
```

匹配

```
abc
```

输出

```
xxx123
```

------

# 五、匹配数字

只保留数字

```bash
echo "abc123xyz" | sed 's/[^0-9]//g'
```

结果

```
123
```

解释

```
^0-9  不是数字
```

------

# 六、删除空行

```bash
sed '/^$/d' file.txt
```

解释

```
^   行开始
$   行结束
```

------

# 七、删除空格行

```bash
sed '/^[[:space:]]*$/d' file.txt
```

匹配

```
空格
tab
空行
```

------

# 八、提取 IP 地址

假设日志：

```
client ip=192.168.1.10
```

提取 IP

```bash
sed -n 's/.*ip=\([0-9\.]*\).*/\1/p'
```

输出

```
192.168.1.10
```

解释

```
\(...\)  分组
\1       引用
```

------

# 九、替换 URL 参数

例如

```
/api/user?id=123&name=tom
```

删除参数

```bash
sed 's/\?.*//'
```

输出

```
/api/user
```

------

# 十、只保留接口路径

日志

```
GET /api/user?id=1 HTTP/1.1
```

提取接口

```bash
sed -n 's/.*GET \(\/api[^ ]*\).*/\1/p'
```

输出

```
/api/user?id=1
```

------

# 十一、去掉字符串前后空格

```bash
sed 's/^ *//;s/ *$//'
```

解释

```
^ *   前空格
 *$   后空格
```

------

# 十二、sed 分组替换

例子

```
2025-03-06
```

变成

```
06/03/2025
```

命令

```bash
sed -E 's/([0-9]{4})-([0-9]{2})-([0-9]{2})/\3\/\2\/\1/'
```

输出

```
06/03/2025
```

------

# 十三、提取接口路径（日志分析常用）

假设日志

```
/api/user?id=123
/api/user?id=456
/api/order?id=1
```

去掉参数

```bash
sed 's/\?.*//'
```

结果

```
/api/user
/api/user
/api/order
```

然后统计

```bash
sed 's/\?.*//' log.txt | sort | uniq -c
```

结果

```
2 /api/user
1 /api/order
```

这其实就是 **接口去重统计的经典方法**。

------

# 十四、sed + grep + awk 常用组合

统计接口调用次数：

```bash
grep "/api" access.log | sed 's/\?.*//' | sort | uniq -c | sort -nr
```

输出

```
500 /api/user
320 /api/order
210 /api/login
```

------

# 十五、sed 高级技巧

删除多行

```
start
aaa
bbb
end
```

删除 start 到 end

```bash
sed '/start/,/end/d'
```

------

# 十六、sed 调试技巧

查看替换结果

```bash
sed -n 's/error/ERROR/p' log.txt
```

只打印修改过的行。

------

# 十七、程序员必背 sed 正则（10条）

```
删除空行
sed '/^$/d'

删除注释
sed '/^#/d'

删除参数
sed 's/\?.*//'

提取数字
sed 's/[^0-9]//g'

去空格
sed 's/^ *//;s/ *$//'

只打印匹配
sed -n '/ERROR/p'

删除范围
sed '/start/,/end/d'

替换全部
sed 's/old/new/g'

修改文件
sed -i 's/old/new/g'

提取IP
sed -n 's/.*ip=\([0-9\.]*\).*/\1/p'
```

------

💡 **结合你刚刚的问题（接口去重统计）**，最实用的一条其实是：

```bash
sed 's/\?.*//'
```

它可以把：

```
/api/user?id=1
/api/user?id=2
/api/user?id=3
```

全部变成：

```
/api/user
```
