# **Linux `head` 命令教程文档**

`head` 命令用于查看文件的前若干行内容，默认显示前 **10 行**。常用于快速预览文件开头内容，如日志、配置文件、脚本等。

------

## **一、基本语法**

```
head [选项] [文件名]
```

- 若不指定文件名，则从标准输入读取内容（可与管道配合使用）。

------

## **二、常用选项**

| 选项          | 作用                             |
| ------------- | -------------------------------- |
| **-n 数字**   | 指定显示行数（默认 10 行）       |
| **-c 字节数** | 指定显示字节数                   |
| **-v**        | 总是显示文件名头部（用于多文件） |
| **-q**        | 不显示文件名头部（用于多文件）   |

------

## **三、常用示例**

### **1. 查看文件前 10 行（默认）**

```
head /var/log/messages
```

------

### **2. 查看文件前 20 行**

```
head -n 20 /etc/passwd
```

或

```
head -20 /etc/passwd
```

------

### **3. 查看前 100 字节内容**

```
head -c 100 test.txt
```

------

### **4. 同时查看多个文件的前几行**

```
head -n 5 file1.txt file2.txt
```

输出中会自动显示类似：

```
==> file1.txt <==
前5行内容...

==> file2.txt <==
前5行内容...
```

------

### **5. 与管道一起使用（结合其他命令）**

#### 查看当前目录下文件列表的前 10 行

```
ls -l | head
```

#### 查看日志输出的前几行

```
dmesg | head -n 15
```

------

## **四、与 `tail` 的区别**

| 命令     | 说明                             |
| -------- | -------------------------------- |
| **head** | 查看文件开头部分                 |
| **tail** | 查看文件末尾部分（默认后 10 行） |

组合使用可快速定位日志：

```
head -n 20 app.log
tail -n 20 app.log
```

------

## **五、实际场景中的使用示例**

### **查看大型文件是否包含正确的头部格式**

例如 CSV 文件：

```
head data.csv
```

### **在分析日志时先看开头是否有异常信息**

```
head /var/log/syslog
```

### **通过 head + grep 查看过滤后结果的前几行**

```
grep "ERROR" app.log | head
```
