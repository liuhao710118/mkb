# 如何根据进程号PID定位具体程序运行的目录

在普通情况下 可以通过ps -ef解密95%的情况 但是往往有那么一些程序 不是从绝对目录启动 参数也不带具体位置信息

那么就要用以下几个命令了



## 查看可执行文件真实路径

```bash
ls -l /proc/{PID}/exe
```

## 查看完整启动命令（含参数）

```bash
cat /proc/1234/cmdline | tr '\0' ' '
```

```bash
ps -p {pid} -o pid,cmd
```





## 进程被删除但仍在运行（排障重点）

### 典型现象

```

ls -l /proc/1234/exe

/proc/1234/exe -> /opt/app/app.bin (deleted)
```

📌 说明：

- 程序文件已被删除或覆盖
- 进程仍在内存中运行

解决方式：

```

# 重启进程释放空间 kill 1234
```
