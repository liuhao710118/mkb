# **Linux `uname` 命令教程文档**

`uname`（Unix Name）用于查看系统内核信息，是运维人员快速了解系统版本、架构、内核等基础信息的常用命令。

------

# **一、基本语法**

```
uname [选项]
```

------

# **二、常用选项说明**

| 选项   | 含义                               |
| ------ | ---------------------------------- |
| **-a** | 显示所有信息（最常用）             |
| **-s** | 显示操作系统名称                   |
| **-r** | 显示内核版本号                     |
| **-v** | 显示内核版本详细信息               |
| **-m** | 显示机器硬件架构（如 x86_64）      |
| **-p** | 显示处理器类型（可能显示 unknown） |
| **-i** | 显示硬件平台                       |
| **-o** | 显示操作系统名称（通常与 -s 一样） |

------

# **三、常用示例**

## **1. 显示所有系统信息（最常用）**

```
uname -a
```

输出示例：

```
Linux server01 4.18.0-348.el8.x86_64 #1 SMP ... x86_64 GNU/Linux
```

包含信息：

| 字段                  | 含义               |
| --------------------- | ------------------ |
| Linux                 | 系统类型           |
| server01              | 主机名             |
| 4.18.0-348.el8.x86_64 | 内核版本           |
| SMP                   | 表示多核处理器支持 |
| x86_64                | 硬件架构           |
| GNU/Linux             | 操作系统           |

------

## **2. 查看操作系统内核名称**

```
uname -s
```

------

## **3. 查看内核版本号（常用于排查兼容性）**

```
uname -r
```

输出例：

```
5.4.0-162-generic
```

------

## **4. 查看详细内核版本信息**

```
uname -v
```

------

## **5. 查看硬件架构（32/64 位）**

```
uname -m
```

输出示例：

```
x86_64
```

------

## **6. 查看处理器类型**

```
uname -p
```

注意：很多系统可能返回 `unknown`

------

## **7. 查看硬件平台**

```
uname -i
```

------

## **8. 查看操作系统类型**

```
uname -o
```

------

# **四、运维中的常用组合命令**

## **1. 判断当前系统是否是 64 位**

```
uname -m
```

返回:

- **x86_64** → 64 位
- **i686/i386** → 32 位

------

## **2. 检查内核版本是否满足软件要求**

例如 Docker 要求内核 `>= 3.10`：

```
uname -r
```

------

## **3. 快速收集系统环境信息**

```
uname -a
```

------

# **五、总结**

`uname` 是非常基础但高频的运维命令，可快速查看：

- 操作系统类型
- 内核版本
- 硬件架构
- 主机信息

最常用命令：

```
uname -a          # 查看系统完整信息
uname -r          # 查看内核版本
uname -m          # 查看架构(32/64bit)
```
