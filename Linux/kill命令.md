# **Linux `kill` 命令教程文档**

`kill` 用于向进程发送信号（signal）。
 最常见用途：**终止进程**、**重新加载配置**、**暂停进程** 等。

Linux 默认支持 64 种信号，进程可以根据不同信号执行不同操作。

------

# **一、基本语法**

```
kill [选项] <PID>
```

常用选项：

| 选项            | 说明               |
| --------------- | ------------------ |
| **-l**          | 显示支持的全部信号 |
| **-s <信号名>** | 指定信号名         |
| **-<信号号>**   | 指定信号编号       |

------

# **二、常用信号（重点掌握）**

| 信号名      | 编号 | 作用                       |
| ----------- | ---- | -------------------------- |
| **SIGTERM** | 15   | 正常终止（默认信号）       |
| **SIGKILL** | 9    | 强制杀死进程，无法拦截     |
| **SIGSTOP** | 19   | 暂停进程（可恢复）         |
| **SIGCONT** | 18   | 恢复被暂停的进程           |
| **SIGHUP**  | 1    | 重新加载配置（常用于服务） |

运维使用最频繁的是 **15（正常停止）** 和 **9（强制停止）**。

------

# **三、常用命令示例**

------

## **1. 正常终止一个进程（推荐方式）**

```
kill 1234
```

等价于：

```
kill -15 1234
```

------

## **2. 强制杀死进程（无法捕获/无法拒绝）**

```
kill -9 1234
```

适用于：

- 程序死锁
- 程序无响应
- SIGTERM 不生效

------

## **3. 查看所有系统信号**

```
kill -l
```

输出示例：

```
1) SIGHUP 2) SIGINT 3) SIGQUIT ... 9) SIGKILL ... 15) SIGTERM ...
```

------

## **4. 给所有匹配的进程名发送信号（pkill）**

```
pkill nginx
```

如：

```
pkill -9 python
```

------

## **5. 给某项服务重新加载配置**

比如 nginx：

```
kill -HUP <nginx-master-PID>
```

等价于 `nginx -s reload`

------

## **6. 暂停一个进程**

```
kill -STOP 1234
```

------

## **7. 恢复被暂停的进程**

```
kill -CONT 1234
```

------

# **四、配套命令**

| 命令         | 说明                        |
| ------------ | --------------------------- |
| **ps**       | 查看进程 PID                |
| **pgrep**    | 根据名称查找 PID            |
| **pkill**    | 根据名称发送信号            |
| **top/htop** | 查看运行中进程并可直接 kill |

------

# **五、实际运维场景示例**

------

## **1. 停止卡死的 Python 程序**

```
kill -9 $(pgrep python)
```

------

## **2. 重启 Nginx 配置而不停止服务**

```
kill -HUP $(cat /run/nginx.pid)
```

------

## **3. 暂停一个 CPU 密集型任务**

```
kill -STOP <PID>
```

------

## **4. 恢复暂停的任务**

```
kill -CONT <PID>
```

------

## **5. 多进程服务强制全部终止**

```
pkill -9 httpd
```

------

# **六、kill 信号说明速查表（常用）**

| 信号名  | 编号 | 作用                 |
| ------- | ---- | -------------------- |
| SIGHUP  | 1    | 重读配置文件         |
| SIGINT  | 2    | 终止 (CTRL+C)        |
| SIGQUIT | 3    | 终止并产生 core dump |
| SIGKILL | 9    | 强制终止             |
| SIGTERM | 15   | 正常终止（默认）     |
| SIGSTOP | 19   | 暂停进程             |
| SIGCONT | 18   | 恢复进程             |

------

# **七、总结**

`kill` 本质上是给进程发送信号，不只是“杀进程”。最常用：

```
kill -15 <PID>     # 正常停止（优雅）
kill -9 <PID>      # 强制停止（暴力）
kill -HUP <PID>    # 重载配置
kill -STOP <PID>   # 暂停
kill -CONT <PID>   # 恢复
```
