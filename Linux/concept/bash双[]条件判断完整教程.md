# Bash `[[ ]]` 条件判断完整教程

> `[[ ... ]]` 是 **Bash 内建关键字**，用于条件判断
> ✔ 更安全
> ✔ 更强大
> ✔ 更易读
> ❌ 非 POSIX（`/bin/sh` 不能用）

------

## 一、基本语法

```bash
[[ 条件表达式 ]]
```

### 在 if 中使用

```bash
if [[ 条件 ]]; then
    commands
fi
```

返回值：

- **真 → exit code 0**
- **假 → exit code 1**

------

## 二、字符串判断（最常用）

### 1️⃣ 相等 / 不相等

```bash
[[ $a == $b ]]
[[ $a != $b ]]
```

> 推荐使用 `==`（更符合 Bash 风格）

------

### 2️⃣ 是否为空

```bash
[[ -z $var ]]   # 为空
[[ -n $var ]]   # 非空
[[ -z $var ]] && echo "empty"
```

------

### 3️⃣ 字符串大小比较（ASCII）

```bash
[[ $a < $b ]]
[[ $a > $b ]]
```

⚠ **注意**

- 在 `[[ ]]` 中 `<`、`>` 不需要转义
- `[ ]` 中必须写成 `\<`

------

## 三、数值判断

```bash
[[ $a -eq 10 ]]
[[ $a -ne 10 ]]
[[ $a -gt 10 ]]
[[ $a -ge 10 ]]
[[ $a -lt 10 ]]
[[ $a -le 10 ]]
```

### 示例

```bash
if [[ $cpu -gt 80 ]]; then
    echo "CPU 高负载"
fi
```

------

## 四、文件与目录判断（运维必会）

| 判断      | 含义     |
| --------- | -------- |
| `-f file` | 普通文件 |
| `-d dir`  | 目录     |
| `-e path` | 存在     |
| `-r file` | 可读     |
| `-w file` | 可写     |
| `-x file` | 可执行   |
| `-s file` | 非空文件 |

```bash
[[ -f /etc/passwd ]] && echo "exists"
```

------

## 五、逻辑组合（强烈推荐）

```bash
[[ 条件1 && 条件2 ]]
[[ 条件1 || 条件2 ]]
[[ ! 条件 ]]
```

### 示例

```bash
[[ -n $ip && $ip =~ ^192\.168\. ]] && echo "内网 IP"
```

------

## 六、模式匹配（glob）

```bash
[[ $var == pattern ]]
```

### 示例

```bash
[[ $file == *.log ]]
[[ $user == root* ]]
```

✔ 支持：

- `*` 任意字符
- `?` 单个字符
- `[]` 字符集合

------

## 七、正则匹配（非常重要 ⭐）

```bash
[[ 字符串 =~ 正则 ]]
```

### 示例

#### 1️⃣ IP 判断

```bash
[[ $ip =~ ^([0-9]{1,3}\.){3}[0-9]{1,3}$ ]]
```

#### 2️⃣ 数字判断

```bash
[[ $var =~ ^[0-9]+$ ]]
```

#### 3️⃣ 邮箱

```bash
[[ $mail =~ ^[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+\.[a-z]{2,}$ ]]
```

### ⚠ 正则使用规则（重点）

- **右侧不要加引号**
- 支持 ERE（扩展正则）
- 捕获组可通过 `BASH_REMATCH` 访问

```bash
if [[ $date =~ ^([0-9]{4})-([0-9]{2})-([0-9]{2})$ ]]; then
    echo "year=${BASH_REMATCH[1]}"
fi
```

------

## 八、变量是否需要引号？

**结论：一般不需要**

```bash
[[ $a == $b ]]   # OK
[[ $a == "" ]]   # OK
```

为什么安全？

- 不会发生 **word splitting**
- 不会发生 **pathname expansion**

## 九、`[[ ]]` 与命令替换

```bash
[[ $(id -u) -eq 0 ]] && echo "root"
```

------

## 十、`[[ ]]` 常见错误（必看）

### ❌ 错误 1：正则加引号

```bash
[[ $ip =~ "^[0-9]+" ]]   # 错
```

### ❌ 错误 2：在 sh 中使用

```sh
#!/bin/sh
[[ -f file ]]   # 报错
```

### ❌ 错误 3：混用 `[ ]`

```bash
[ $a == 1 ]   # 不标准
```

------

## 十一、实战模板（推荐收藏）

### 1️⃣ 参数校验

```bash
if [[ $# -ne 1 || ! $1 =~ ^[0-9]+$ ]]; then
    echo "Usage: $0 <number>"
    exit 1
fi
```

------

### 2️⃣ 服务状态检测

```bash
if [[ -f /var/run/nginx.pid && -s /var/run/nginx.pid ]]; then
    echo "nginx running"
fi
```

------

### 3️⃣ IP 白名单

```bash
if [[ $ip == 10.* || $ip == 192.168.* ]]; then
    echo "internal"
fi
```

------

## 十二、最佳实践（工程级）

✅ Bash 脚本头部

```bash
#!/usr/bin/env bash
set -euo pipefail
```

✅ 条件判断统一用 `[[ ]]`

✅ 复杂判断用 `[[ ... && ... ]]`，不用 `-a -o`
